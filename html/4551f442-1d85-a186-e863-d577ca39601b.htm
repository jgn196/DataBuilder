AppendByte
Adds a byte to the build "recipe".
AppendBytes
Adds an arbitary chunk of binary data to the build "recipe".
AppendInt16
Adds a 16-bit signed integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
AppendInt32
Adds a 32-bit signed integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
AppendInt64
Adds a 64-bit signed integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
AppendSByte
Adds a signed byte to the build "recipe".
AppendText
Adds a string to the build "recipe". The string will not be null terminated. If the SetEncoding method has not been called, encoding defaults to ASCII.
AppendUInt16
Adds an unsigned 16-bit integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
AppendUInt32
Adds an unsigned 32-bit integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
AppendUInt64
Adds an unsigned 64-bit integer to the build "recipe". If the SetByteOrder method has not been called, little endian is used by default.
Build
Builds the data according to the "recipe" that has been built up by calling methods on this object.

Repeat
Adds the result of a secondary DataBuilder as a repeating pattern. Passing a DataBuilder itself as the repeating pattern is illegal.
SetByteOrder
Sets the byte order (endianness) to use when appending integers. Any integers that have already been appeneded are unaffected.
SetEncoding
Sets the encoding to use when appending text strings. Any text strings that have already been appended are unaffected.
